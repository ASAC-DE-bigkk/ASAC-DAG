# 작업 계획 (Plans)

개별 작업 단위의 계획 문서를 모으는 폴더. 전체 로드맵은 [../plan.md](../plan.md) 참고.

## 컨벤션

- **1 작업(이슈/브랜치) = 1 계획 문서.** 코드 착수 전에 여기에 계획을 먼저 작성·합의한다.
- 파일명: `YYYY-MM-DD-<slug>.md`
  - 날짜 = 계획 작성일, slug = 브랜치명과 대응 (예: 브랜치 `feat/70-common-http` → `2026-07-05-feat-70-common-http.md`)
  - 이슈 번호가 아직 없으면 slug에서 생략하고, 이슈 생성 후 파일명에 반영한다.
- 계획이 확정되면 상태를 `합의됨`으로 바꾸고 작업을 시작한다. 작업 완료 후 결과가 계획과 달라졌으면 문서에 반영한다.

## 템플릿

```markdown
# <작업 제목>

- 상태: 초안 | 합의됨 | 진행 중 | 완료
- 작성일: YYYY-MM-DD
- 이슈: #NN / 브랜치: feat/NN-<slug>

## 배경 · 목표

## 범위 (in / out)

## 계획

1. ...

## 리스크 · 열어둔 질문
```
